**핵심 분업 원칙 (Fable 스프린트에서 검증된 것):**
1. **판정과 적용을 분리** — 에이전트는 판단만 출력(JSON), 파일 조작은 스크립트가. 사고가 나면 스크립트만 고치면 되고 롤백이 깨끗하다.
2. **배치 사이클 고정** — 목록 산출 → 워크플로 → post_batch_fix → validate_canon → index 재생성 → 커밋·푸시. 이 순서를 깨지 말 것 (§3 상세).
3. **한도 사고 대응** — 세션/주간 한도로 워크플로가 죽어도 산출물은 파일로 남는다. git status 실측 → 체크포인트 커밋 → resume. 캐시 미적중 시 잔여 목록만 재기동.
4. **검수 불신 원칙** — 서브에이전트의 파괴적 주장(내용 소실·"X를 Y로 통일해야")은 git·원본 대조 후에만 수용. 과교정 사례: 소환 7조약→5등급 (§6 함정 3).
