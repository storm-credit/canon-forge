## 3. 배치 사이클 표준 절차 (모든 대량 작업 공통)

1. **대상 목록 산출** (기계 스캔) → scratchpad JSON
2. **워크플로 기동** — 판정·집필은 에이전트, **적용은 기계 스크립트** 분리 원칙
3. 완료 통지 → `post_batch_fix.py` → `validate_canon.py` → **폴더 index 재생성** → 커밋·푸시
4. **세션/주간 한도로 죽으면**: 산출물은 파일로 남아 있음 — `git status`로 실측 후 체크포인트 커밋 → resume 또는 잔여 목록 재기동
5. **검수의 파괴적 주장(내용 소실·깨짐)은 git 대조 후에만 수용** — 오탐 전례 다수
