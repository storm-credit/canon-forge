## §3 운영 불변식 (감사 실증 교훈)

1. **반박 검증 의무** — 모든 감사 발견은 건당 반박관이 원문 재대조 후 확정 (실측 기각률 40~54% — 안 거르면 과잉 수선이 원문 훼손).
2. **문서 단위 무결에 에이전트 금지** — 오탈자·스텁·깨진 문자는 기계 전수가 0비용 담당.
3. **발견→수선→규정 승격** 사이클 — 같은 유형이 재발하지 않도록 게이트 금지어·집필 계약·표기 지침으로 승격 (방각→금지어 24종 체제, 화 경계 재고정 원칙 등).
4. **발명 0 게이트** — 수선은 기존 캐논 조립·행 단위 수술만. 수치·명명 신설은 "작가 배정 대기" 칸 또는 [작가 판단] 태그. 위임("개연성 맞게") 시에도 근거·되돌림 경로 명기.
5. **원문 보존** — 삭제 대신 봉합 배너·주석·이력 노트 (append-only 우선). 중복 문서는 삭제 대신 "정본 참조" 스텁.
6. **Obsidian 간섭 방어** — 대량 편집 전 종료 요청, 후 git diff 행수 검사.
7. 점수의 의미 — 렌즈 점수는 절대값이 아니라 **다음 수선의 좌표**다. 재채점은 같은 기준·같은 편성으로만 비교.
