# 💎 Diamond-Class Novel AI System Architecture

> *"에픽 판타지의 짐승들은 단순한 도구가 아니라, 전장을 함께 누비며 제국의 흥망성쇠를 목도하는 **'장대한 대자연의 동반자'**들이다. AI는 이 거대한 서사와 전우애를 완벽하게 조율하는 **'Narrative Graph DB 오케스트레이터'**여야 한다."*

## 1. Core Philosophy (핵심 철학)
본 아키텍처는 [완전 자동 생성(Full-Auto)]이 발생시키는 '개연성 붕괴'와 '공장형 서사'를 원천 차단하는 **[반자동(Semi-Automatic) 코-라이터(Co-Writer)]** 시스템이다.
인간 작가(주조종사)가 **'톨킨 반지의 제왕, 드래곤 라자, 하얀 늑대들'**과 같은 장엄하고 낭만적인 에픽 하이 판타지의 서사와 동료애를 지휘하면, AI 에이전트는 기저에서 수백 개의 세계관 설정 충돌을 방어하고 핍진성(거대 괴수 사육의 현실적 군수/병참 문제 등)의 뼈대를 묵직하게 유지하는 엔진 역할을 수행한다.

## 2. The 5-Layer System Framework (5계층 작가 자동화 시스템)

| Layer | 명칭 | 기능 (AI의 역할) | 작가의 역할 |
| :--- | :--- | :--- | :--- |
| **A** | **Story Kernel Layer** | 작품의 대주제(Theme), 핵심 갈등(Core Conflict), 세계의 법칙(World Rule)을 불변의 메타데이터로 보존. | 에픽 시대관 주입 및 영웅들의 방향성 제시 |
| **B** | **Plot Engine Layer** | 정통 판타지에 맞춘 웅장한 골조 제공. (왕도의 서막, 3막, 무훈시적 영웅 여정) | 장대한 플롯 전개 및 챕터별 대서사시 발동 |
| **C** | **Character Agent Layer** | 각 영웅을 독립된 에이전트로 취급. 성격, 가치관, 과거의 맹세와 기억 할당. | 인물 간의 끈끈한 유대감 및 대화 톤 최종 검수 |
| **D** | **Narrative Graph Layer** | **가장 중요한 계층**. 국가, 맹수, 기사단을 그물망처럼 연결해 세계관(방대한 대륙력)의 일관성 유지. | 역사적 서사와 상징, 전설의 은유적 연출 |
| **E** | **Scene Generator Layer** | 생성 엔진. A~D 레이어의 통제를 받으며 대규모 전투, 웅장한 자연, 동료 간의 서사를 렌더링. | 결과물의 문학적 첨삭 및 낭만주의적 톤 수정 |

## 3. Project-Specific Plot Engine — 잊힌 소환사

**초점: 불쾌한 고문/호러가 아닌, '대서사시적 장엄함'과 '야성을 기르는 거친 현실성'의 조화**

1. **장대한 서사 엔진 (Epic Saga Engine)** — 종족 간의 운명, 제국의 흥망, 대자연과의 맹세가 얽힌 서사구조. 짐승의 야성이 기사의 생명과 직결되며, 한 시대의 맹수가 다음 세대 영웅에게 전승되는 신화적 연속성.
2. **역사서 / 기록 문서 구조 (Chronicle + Document Protocol)** — 고대 도서관의 타자기록, 근위 기사단의 장엄한 맹세문, 제국군의 웅장한 전술 교범 등 깊이 있는 역사적 문서 교차 서술.
3. **심볼 웹 (Symbol Web: 맹세/긍지/투쟁)** — 코스믹 호러/광기 요소를 일체 소거하고, 인간의 **'위대한 투쟁'**, 대자연 맹수와 교감하는 **'긍지(Pride)'**, 시대를 관통하는 진중한 **'맹세(Oath)'**를 주요 문학 상징으로 사용.
